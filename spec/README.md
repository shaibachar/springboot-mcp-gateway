# Specification Overview

Use these specs to understand the MCP gateway at a glance. Each file keeps the focus on what matters for implementers and reviewers.

- `design.md`: Core goals, behaviors, and compatibility notes.
- `feature_structured_errors.md`: Error contract and required response fields.
- `feature_discovery_caching.md`: Discovery cache rules and refresh expectations.
- `feature_graphql_metadata.md`: Required metadata for GraphQL and REST tool schemas.
- `testing.md`: Minimal checks to verify the contract.
